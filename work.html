Viito

Rails Application

Description

Team Stream was is an application that allows anyone to see what twitter has to say about their sports team. This was a collaboration project with one other colleague and out vision was to tap into the Twitterverse to see what the pleasant twitterers around the world had to say about our sports team. We used the Twitter API to return tweets that contained our search parameter (a team's name), then we ran it through Alchemy API to determine the sentiment of that tweet. We then translated this data in form of Javascript visuals such as charts, positivity gauges, and streaming tweets. The visuals are still in need of some work, but this was a great learning experience.

This was my first attempt at using APIs in a project. This was a frustrating, but good learning experience with using APIs, collaborating on git, and overcoming issues. The final app is still not exactly what me and my partner envisioned, but we will be updating this in the near future.

Tools

Ruby on Rails, Heroku, Git, Github, Spotify API, Echo Nest API, Html, CSS, Illustrator

Polyjott

Evernote Hackathon at Cal Poly

Description

At a Hackathon at Cal Poly hosted by Evernote our team came up with a web application that would help students in school. The idea was to create a web application where students could upload and share their class notes using Evernote. Once a student uploads his or her notes, other student would be able to view and vote on the quality of notes. We came up with this idea when thinking of those emergencies when missing class would be inevitable and creating an application where quality notes could be found. For this project I worked on the front end of our site. This was my first time using Bootstraps to create a project. I also used Photoshop and Illustrator to create graphics and illustrations for the site. We never did complete the project to production, but I used it as a very helpful learning experience.

Tools

Photoshop, Illustrator, Html, CSS, Coda, Twitter Bootstraps
